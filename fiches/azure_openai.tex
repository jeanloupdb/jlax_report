\section*{En bref}
Azure OpenAI est la version souveraine de l'API OpenAI hébergée dans les datacenters Microsoft en Union Européenne. Fonctionnellement identique à l'API OpenAI classique (mêmes modèles GPT-4o, GPT-4o-mini), elle offre une garantie RGPD stricte : les données traitées ne quittent jamais l'UE et ne sont pas utilisées pour l'entraînement des modèles. À utiliser pour les clients sensibles : cabinets juridiques, médicaux, RH, finance.

\begin{pointcle}
  Azure OpenAI est un \textbf{argument commercial fort} avec les grands comptes et les secteurs réglementés. La différence de prix vs OpenAI standard est marginale (+10–20\%) pour une conformité RGPD garantie contractuellement par Microsoft.
\end{pointcle}

\section{Ce que voit le client}

\begin{clientbox}
  \textit{« J'ai les mêmes capacités IA qu'OpenAI — résumés, rédaction, analyse — mais mes données ne quittent jamais l'Union Européenne. Mon avocat et mon DPO ont validé l'usage. Je peux mettre en place l'IA sur nos dossiers clients sensibles sans risque juridique. »}
\end{clientbox}

\textbf{Gains mesurables :}
\begin{itemize}
  \item \textbf{Conformité RGPD} stricte et contractuelle avec Microsoft
  \item Argument de confiance décisif pour les clients grands comptes et secteurs réglementés
  \item Mêmes performances que l'API OpenAI — \textbf{zéro compromis technique}
  \item Traitement possible de données personnelles, médicales, juridiques
\end{itemize}

\section{Installation technique}

\textbf{Étape 1 — Créer la ressource Azure OpenAI :}
\begin{lstlisting}[style=terminal, caption={Azure Portal — Création de la ressource}]
# Azure Portal → Créer une ressource → Azure OpenAI
# Paramètres :
#   Subscription: Votre abonnement Azure
#   Resource group: rg-jlax-client-nom
#   Region: France Central (ou Sweden Central pour EU)
#   Name: jlax-client-nom-openai
#   Pricing tier: Standard S0
# → Valider et attendre la création (2-5 minutes)
\end{lstlisting}

\textbf{Étape 2 — Déployer un modèle :}
\begin{lstlisting}[style=terminal]
# Azure OpenAI Studio (oai.azure.com) → Deployments → Create
# Model: gpt-4o-mini (recommandé) ou gpt-4o (tâches complexes)
# Deployment name: gpt4omini-prod  (note: pas de points dans le nom)
# Tokens Per Minute Rate Limit: 100K TPM (ajuster selon usage)
\end{lstlisting}

\textbf{Étape 3 — Récupérer les identifiants :}
\begin{lstlisting}[style=terminal]
# Azure OpenAI Studio → Settings → Resource
# Endpoint: https://jlax-client-nom-openai.openai.azure.com/
# API Key: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx (Keys and Endpoint)
# Deployment name: gpt4omini-prod
# API Version: 2024-02-01 (vérifier la dernière version stable)
\end{lstlisting}

\textbf{Étape 4 — Configurer dans n8n :}
\begin{enumerate}
  \item n8n → Credentials → Add → \textbf{Azure OpenAI}
  \item Resource Name : \texttt{jlax-client-nom-openai}
  \item API Key : coller la clé Azure
  \item Dans les nodes OpenAI : sélectionner \textbf{Azure} comme provider
  \item Deployment Name : \texttt{gpt4omini-prod}
\end{enumerate}

\textbf{Utilisation dans Flowise :}
\begin{itemize}
  \item Flowise → Credentials → Add → \textbf{Azure ChatOpenAI}
  \item Renseigner les mêmes paramètres : Resource Name, API Key, Deployment Name
  \item Tous les assistants RAG peuvent basculer d'OpenAI à Azure en changeant la credential
\end{itemize}

\section{Rendre invisible côté client}

\begin{itemize}
  \item Le client ne voit pas la différence entre OpenAI et Azure OpenAI — même résultats
  \item L'argument de vente : « \textbf{nos IA sont hébergées en France} » — fort auprès des directions
  \item La facturation Azure OpenAI est incluse dans la facture mensuelle Jlax
  \item Basculement OpenAI → Azure OpenAI : changer uniquement la credential dans n8n/Flowise
\end{itemize}

\end{checklist}
